# MindForge 功能页面原型设计（中文）

> 目标：基于已完成 PRD 与商业计划，将 MindForge 的 5 大核心模块拆解为可直接用于高保真设计与前端开发的中保真原型说明。  
> 设计原则：结构化思考训练、低门槛高反馈、可量化成长、AI 陪练闭环。

---

## 0. 全局设计规范（适用于 5 大模块）

### 0.1 信息架构（IA）总览

```text
MindForge
├─ 思维体检
│  ├─ 体检首页
│  ├─ 评估进行页
│  └─ 评估报告页
├─ 每日训练
│  ├─ 今日任务页
│  ├─ 训练执行页
│  └─ 训练结果页
├─ AI 对练与反馈
│  ├─ 对练大厅
│  ├─ 对练会话页
│  └─ AI 反馈页
├─ 决策与复盘系统
│  ├─ 决策工作台
│  ├─ 决策记录详情
│  └─ 复盘分析页
└─ 成长面板
   ├─ 成长总览
   ├─ 能力雷达/趋势
   └─ 成长计划页
```

### 0.2 全局 UI 组件库（建议）
- 顶部导航（Logo、模块切换、通知、个人中心）
- 左侧二级导航（桌面端）/ 底部 Tab（移动端）
- 卡片（数据卡、任务卡、反馈卡）
- Stepper（流程步骤）
- 图表组件（折线、雷达、柱状、热力图）
- AI 助手抽屉（右侧浮层）
- Toast/Modal（反馈与确认）

### 0.3 响应式原则
- **桌面端**：12 栅格，信息密度高，支持分屏（主内容 + AI 辅助面板）
- **移动端**：单列流式，关键 CTA 固定底部，复杂图表转卡片摘要 + 可展开详情

## 模块 1：思维体检（Thinking Assessment）

### 页面 1-1：体检首页（入口与引导）

**页面结构（线框）**
```text
[Top Nav]
[欢迎区：本次体检目标 + 预计 12 分钟]
[能力维度卡片 x6：理解力/结构力/判断力/反思力/迁移力/表达力]
[开始体检 CTA]
[历史记录入口]
```

- 核心交互：开始体检、查看维度说明、选择“快速版/完整版”
- 信息架构：目标说明 > 评估维度 > 测评选项 > 历史入口
- UI 特点：大卡片 + 渐进引导，降低首次使用门槛
- 用户流程：进入首页 → 阅读说明 → 选择版本 → 开始评估

### 页面 1-2：评估进行页（题目与作答）

**页面结构（线框）**
```text
[进度条 35%][剩余时间]
[题干区]
[作答区：单选/多选/排序/简答]
[AI 提示按钮：解释题意，不给答案]
[上一步] [下一题/提交]
```

- 核心交互：作答、草稿暂存、题意解释、标记稍后再答
- 信息架构：当前题目 > 作答输入 > 辅助工具 > 流程控制
- UI 特点：无干扰模式（隐藏多余导航），突出题目与输入区
- 用户流程：阅读题目 → 作答 → 下一题 → 完成提交

### 页面 1-3：评估报告页（结果与建议）

**页面结构（线框）**
```text
[总分与等级]
[能力雷达图]
[维度详情卡：强项/短板/证据]
[个性化建议：推荐训练路径]
[按钮：去每日训练 / 导出报告]
```

- 核心交互：查看分维度解释、切换时间对比、导出 PDF
- 信息架构：总览 > 分析 > 建议 > 下一步行动
- UI 特点：数据可视化 + 可执行建议同屏呈现
- 用户流程：查看结果 → 理解短板 → 一键进入训练计划

## 模块 2：每日训练（Daily Training）

### 页面 2-1：今日任务页

- 页面结构：今日目标条、任务清单（3~5 项）、预计时长、完成进度
- 核心交互：开始任务、跳过并说明原因、调整任务顺序
- 信息架构：今日目标 > 任务拆解 > 状态追踪
- UI 特点：任务卡片带“价值标签”（高价值/高紧急）
- 用户流程：查看今日计划 → 选择任务 → 进入训练执行页

### 页面 2-2：训练执行页

- 页面结构：题目/案例区、思维模板区（事实-解释-情绪、证据框架）、输入区
- 核心交互：按模板填写、保存草稿、请求 AI 提示、提交训练
- 信息架构：训练内容 > 思考框架 > 输出结果
- UI 特点：左右分栏（桌面）/ 上下折叠（移动）
- 用户流程：阅读任务 → 使用模板完成 → 提交

### 页面 2-3：训练结果页

- 页面结构：本次评分、AI 点评、改进建议、下一任务推荐
- 核心交互：查看逐项反馈、重做、加入错题本
- 信息架构：结果总览 > 问题定位 > 下一步建议
- UI 特点：红黄绿反馈标签，突出可改进点
- 用户流程：接收反馈 → 修正认知 → 继续下一轮训练

## 模块 3：AI 对练与反馈（AI Sparring & Feedback）

### 页面 3-1：对练大厅

- 页面结构：对练模式卡（辩论/决策/复盘/表达）、难度选择、场景库
- 核心交互：选择模式、选择角色（面试官/导师/质疑者）、开始对练
- 信息架构：模式选择 > 参数配置 > 启动会话
- UI 特点：场景标签（职场/学习/创业/日常）
- 用户流程：选模式 → 设定场景 → 进入会话页

### 页面 3-2：对练会话页

**页面结构（线框）**
```text
[会话头部：模式/难度/剩余轮次]
[消息流：用户 vs AI]
[右侧：实时评分(逻辑/证据/结构/表达)]
[输入框 + 语音输入 + 引用资料]
[结束对练]
```

- 核心交互：文本/语音输入、引用证据、请求“更强反驳”
- 信息架构：会话过程 > 实时评分 > 输入输出
- UI 特点：评分实时刷新，促进即时纠偏
- 用户流程：多轮对练 → 收敛观点 → 结束并生成反馈

### 页面 3-3：AI 反馈页

- 页面结构：本场总结、关键错误、认知偏差检测、强化练习推荐
- 核心交互：跳转相关训练、收藏反馈、分享给教练/同伴
- 信息架构：总结 > 诊断 > 练习建议
- UI 特点：偏差标签（确认偏误/过度自信/情绪推理）
- 用户流程：查看诊断 → 领取针对性练习 → 继续训练闭环

## 模块 4：决策与复盘系统（Decision & Retrospective）

### 页面 4-1：决策工作台

- 页面结构：问题定义区、方案 A/B/C、收益成本矩阵、风险评估
- 核心交互：填写决策模板、设置“验证时间点”、提交决策日志
- 信息架构：问题定义 > 方案比较 > 风险与假设 > 决策输出
- UI 特点：结构化表单 + 决策矩阵可视化
- 用户流程：定义问题 → 比较方案 → 记录决策

### 页面 4-2：决策记录详情

- 页面结构：当时判断、证据来源、置信度、预期结果、当前状态
- 核心交互：更新实际结果、标注偏差、发起复盘
- 信息架构：历史决策快照 > 当前结果 > 偏差分析入口
- UI 特点：时间轴组件展示“预测 vs 现实”
- 用户流程：回看决策 → 补全结果 → 进入复盘

### 页面 4-3：复盘分析页

- 页面结构：复盘五问（目标/判断/方法/执行/节奏）、根因图、改进动作
- 核心交互：选择错误类型、生成改进清单、设置下次提醒
- 信息架构：结果回顾 > 根因定位 > 系统修正
- UI 特点：因果链可视化（鱼骨图/树状图）
- 用户流程：拆解失败原因 → 制定修正动作 → 进入后续训练

## 模块 5：成长面板（Growth Dashboard）

### 页面 5-1：成长总览

- 页面结构：成长分、连续训练天数、里程碑、今日建议
- 核心交互：切换周/月/季度、查看里程碑详情
- 信息架构：总体状态 > 最近进展 > 下一步目标
- UI 特点：高激励视图（连胜、勋章、阶段升级）
- 用户流程：查看进展 → 明确下一步训练重点

### 页面 5-2：能力雷达与趋势页

- 页面结构：6 维能力雷达、训练频率趋势、决策命中率趋势
- 核心交互：按能力筛选、对比历史阶段、查看关键事件标注
- 信息架构：能力横截面 > 时间趋势 > 事件关联
- UI 特点：多图联动，点击点位可跳转到具体记录
- 用户流程：识别增长/停滞维度 → 回看证据 → 调整计划

### 页面 5-3：成长计划页

- 页面结构：90 天目标、周目标拆解、习惯追踪、风险预警
- 核心交互：编辑计划、调整强度、订阅提醒、与 AI 共创计划
- 信息架构：目标设定 > 路径拆解 > 执行追踪
- UI 特点：甘特式周计划 + 习惯打卡组件
- 用户流程：设定目标 → 分解行动 → 每周校准

## 桌面端 / 移动端关键差异（可直接用于响应式设计稿）

| 设计点 | 桌面端 | 移动端 |
|---|---|---|
| 导航 | 顶部 + 左侧二级导航 | 底部 Tab + 顶部返回 |
| 页面布局 | 双栏/三栏并行 | 单栏卡片流 |
| AI 助手 | 右侧常驻抽屉 | 底部半屏弹层 |
| 图表展示 | 全量图表+筛选面板 | 概览卡片 + 详情抽屉 |
| 关键操作 | 页面内多按钮 | 固定底部主 CTA |
| 表单输入 | 多区块同屏编辑 | 分步填写（Stepper） |

## 端到端用户流程（总览）

```text
首次进入 → 思维体检 → 生成能力画像 → 每日训练
→ AI 对练强化 → 决策实践与复盘 → 成长面板追踪
→ 动态调整训练计划 → 周期性复测
```

## 给 UI/UX 与前端的落地说明

1. **优先级**：先做 1-1、2-2、3-2、4-1、5-1（最小闭环）
2. **设计系统**：建议先统一 Token（颜色、字体、间距、圆角、阴影）
3. **组件化开发**：卡片、图表容器、评分条、模板输入区优先抽象
4. **埋点建议**：开始率、完成率、重做率、对练轮次、复盘触发率
5. **可访问性**：保证键盘操作、色弱友好、文本对比度达标
